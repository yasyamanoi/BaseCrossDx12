BaseCrossDx12ドキュメント

【Sample001】単純な3D空間の描画

　このサンプルはSample001ディレクトリ内にあります。VisualStdioで該当ソリューション（VS2022でBaseCrossDx12VS2022.sln）を開いてください。
　ビルド後実行すると、以下の画面が現れます。
シーンクラス（Scene.h、Scene.cpp）
　ソリューションエクスプローラを見るとBaseCrossDx12フィルタの中にソース ファイルフィルタ、およびヘッダー ファイルフィルタに、いくつかのファイルがあります。これらを記述することで、上記のような画面が表示されます。
　表示しているだけで、コントローラやキーボードに反応するわけでもありません。壁模様のオブジェクトが行ったり来たりしています、Escキーで終了させることができます。
　Scene.h、Scene.cppはシーンクラスです。シーンクラスは、表示される画面そのものと思っていただいても差し支えありません。このクラスは、ライブラリ中BaseSceneクラスを継承して作られます。
　BaseSceneクラスは、Dx12の大部分の制御を行うクラスです。かなり複雑な内容ですが、Sceneクラスでは、このサンプルで最低限必要な処理のみ多重定義して実装します。
　以下がScene.hのSceneクラスの宣言部です。 
	//--------------------------------------------------------------------------------------
	// シーン
	//--------------------------------------------------------------------------------------
	class Scene : public BaseScene
	{
		vector<shared_ptr<MyObject>> m_myObjectvec;
	public:
		Scene(UINT frameCount, PrimDevice* pSample);
		virtual ~Scene();
		template<typename T, typename... Ts>
		shared_ptr<T> AddMyObject(ID3D12GraphicsCommandList* pCommandList, Ts&&... params) {
			try {
				auto Ptr = ObjectFactory::Create<T>(pCommandList, params...);
				m_myObjectvec.push_back(Ptr);
				return Ptr;
			}
			catch (...) {
				throw;
			}
		}

		//アクセサ
		std::shared_ptr<PerspecCamera> GetMyCamera() const {
			return m_myCamera;
		}
		std::shared_ptr<LightSet> GetLightSet() const {
			return m_myLightSet;
		}
	protected:
		std::shared_ptr<PerspecCamera> m_myCamera;
		std::shared_ptr<LightSet> m_myLightSet;

		virtual void CreateAssetResources(ID3D12Device* pDevice, ID3D12GraphicsCommandList* pCommandList)override;
		virtual void UpdateConstantBuffers()override;
		virtual void CommitConstantBuffers()override;

		virtual void Update(double elapsedTime)override;

		virtual void ShadowPass(ID3D12GraphicsCommandList* pCommandList)override;
		virtual void ScenePass(ID3D12GraphicsCommandList* pCommandList)override;
	};
　protectedメンバーの中にいくつかの多重定義された関数があります。これらを記述することで、シーン内に必要なオブジェクトを表示することができます。
　これらの多重定義された関数は、どこかで呼び出す必要がありません。システムで必要なタイミングで呼び出されます。
CreateAssetResources()関数
　CreateAssetResources()関数は、表示すべきオブジェクトを作り出すときに呼び出されます。
　実装は以下のようになっています。 
	void Scene::CreateAssetResources(ID3D12Device* pDevice, ID3D12GraphicsCommandList* pCommandList)
	{
		m_myCamera = shared_ptr<PerspecCamera>(new PerspecCamera());
		m_myLightSet = shared_ptr<LightSet>(new LightSet());

		TransParam param;
		param.scale = XMFLOAT3(1, 1, 1);
		param.rotOrigin = XMFLOAT3(0, 0, 0);
		auto quat = XMQuaternionIdentity();
		param.quaternion = bsm::makeF4(quat);
		param.position = XMFLOAT3(0.0, 5.0, 0.0f);
		AddMyObject<WallBox>(pCommandList, param);
		param.scale = XMFLOAT3(50, 1, 50);
		param.position = XMFLOAT3(0, -0.5, 0.0f);
		AddMyObject<SkyStage>(pCommandList, param);
	}
　CreateAssetResources()関数をはじめ、BaseSceneクラスから継承する関数は、こう書かなければいけない、というわけでもなく、各オブジェクトを効率よく管理できる設計であれば、サンプル通りでなくても構いません。
　このサンプルでは、あらかじめSceneクラスに 
		vector<shared_ptr<MyObject>> m_myObjectvec;
　というMyObjectの配列を持っており、それを使って各オブジェクトを管理します。
　MyObjectクラスはMyObject.h/cppに記述されています。
CreateAssetResources()関数では、最初に 
		m_myCamera = shared_ptr<PerspecCamera>(new PerspecCamera());
		m_myLightSet = shared_ptr<LightSet>(new LightSet());
　という形でカメラとライトを作成しています。
　その後WallBoxクラスとSkyStageクラスの、2つのインスタンスをAddMyObject()関数を使って作成しています。AddMyObject()関数はテンプレート関数になっていて、Scene.hに記述されている通り 
		template<typename T, typename... Ts>
		shared_ptr<T> AddMyObject(ID3D12GraphicsCommandList* pCommandList, Ts&&... params) {
			try {
				auto Ptr = ObjectFactory::Create<T>(pCommandList, params...);
				m_myObjectvec.push_back(Ptr);
				return Ptr;
			}
			catch (...) {
				throw;
			}
		}
　という記述です。BaseCross64を使ったことがある人はわかると思いますが、MyObjectクラスはObjectInterfaceの派生クラスになっており、AddMyObject()関数を使うことで、MyObject::OnCreate()関数を自動的に呼び出すことができます。
計算ライブラリ
　BaseCross64を使った人はわかるかと思いますが、BaseCross64では、計算ライブラリとしてbsm::Vec3、bsm::Vec4クラスなどがありました。これらはDirectXMathをラッピングするクラスで、DirectXMathのクラスを渡しても動くようになっていたのですが、BaseCrossDx12では、これらのクラスはなくしてDirectXMathだけで計算するようにしました。
　とはいえDirectXMathは多機能なのですが、ほとんど関数呼び出しで記述しなければならず、また、その根っこにあるXMVECTORやXMMATRIXは使うのに結構厄介だったりするので、BaseCrossDx12では、汎用的な型はXMFLOAT3やXMFLOAT4X4などを使うようにしています。ただそうすると計算関数を呼ぶためにXMVECTORやXMMATRIXに、いったん変換しなければいけません。ですのでそういった変換処理をする関数などをbsmネームスペースに記述しています。Scene::CreateAssetResources()関数におけるbsm::makeF4()関数などがそうです。この関数はXMVECTORからXMFLOAT4を作り出します。
MyObjectクラス
　Scene::AddMyObject()関数によって追加されるMyObjectはMyObject.h/cppに記述されています。
　MyObjectクラスのコンストラクタはprotectedメンバになっており、直接は作成できません。
　派生クラスを作り、それをScene::AddMyObject()関数で追加します。以下はWallBoxクラスの宣言部です。 
	class WallBox : public  MyObject {
		double m_totalTime;
	protected:
	public:
		WallBox(const TransParam& param);
		virtual ~WallBox();
		virtual void OnCreate(ID3D12GraphicsCommandList* pCommandList);
		virtual void OnUpdate(double elapsedTime);
	};
　コンストラクタの引数はTransParam型となっており、これは構造体です。BaseMath.hに記述されています。world行列を作り出すのに便利な構造体です。 
	struct TransParam {
		XMFLOAT3 scale;
		XMFLOAT3 rotOrigin;
		XMFLOAT4 quaternion;
		XMFLOAT3 position;
	};
　bsmネームスペースには、この構造体を渡すとworld行列を作り出すbsm::affineTransformation()関数が用意されています。
OnCreate()関数
　WallBox::OnCreate()関数は、以下のような記述です。 
	void WallBox::OnCreate(ID3D12GraphicsCommandList* pCommandList){
		MyObject::OnCreate(pCommandList);
		//メッシュ
		m_mesh = BaseMesh::CreateCube(pCommandList, 1.0f);
		//テクスチャ
		auto texFile = App::GetRelativeAssetsDir() + L"wall.jpg";
		m_texture = BaseTexture::CreateTextureFlomFile(pCommandList, texFile);
	}
　関数の先頭に 
		MyObject::OnCreate(pCommandList);
　という記述がありますが、これは親クラス（MyObjectクラス）のOnCreate()関数を呼び出しています。
　MyObject::OnCreate()関数は、WallBoxクラス、SkyStageクラス共通の処理をまとめてあります。最初にその関数を呼び出しています。以下のような記述になっています。 
	void MyObject::OnCreate(ID3D12GraphicsCommandList* pCommandList) {
		auto pBaseScene = BaseScene::Get();
		auto& frameResources = pBaseScene->GetFrameResources();
		auto pBaseDevice = BaseDevice::GetBaseDevice();
		//デフォルトのコンスタントバッファ
		for (size_t i = 0; i < BaseDevice::FrameCount; i++) {
			m_constantBufferIndex = 
				frameResources[i]->AddBaseConstantBufferSet<BaseConstantBuffer>(pBaseDevice->GetD3D12Device());
		}
		//シャドウマップのコンスタントバッファ
		for (size_t i = 0; i < BaseDevice::FrameCount; i++) {
			m_shadowConstantBufferIndex =
				frameResources[i]->AddBaseConstantBufferSet<ShadowConstant>(pBaseDevice->GetD3D12Device());
		}
	}
　ここではデフォルトのコンスタントバッファ（シーン用）とシャドウマップのコンスタントバッファを作り出しています。
　ここでFrameResourcesという新しい概念が出てきました。Dx12プログラムではバックバッファとフロントバッファという複数のレンダーターゲットをを持っています。BaseCrossDx12では、3つのバッファを持ち、ターンごとの描画が終了するたびに、バックバッファとフロントバッファを入れ替えます。3つあるので、つねにバックバッファ状態のは2つあるということです（描画するのはそのうちどちらかですが。）
　FrameResourcesはそれぞれのバッファに対応します。バッファが入れ替わっても、GPU側コンスタントバッファへの書き込みができるようにBaseDevice::FrameCountの数だけ（すなわち3つ）、GPU側コンスタントバッファを作成します。
　AddBaseConstantBufferSet()テンプレート関数は、GPU側コンスタントバッファと書き込みオンリーのバッファを作成し、シェーダーに渡せるようにします。
　戻り値が、m_constantBufferIndex（シーン用）、m_shadowConstantBufferIndex（シャドウマップ用）に上書きされながらループしますが、フレームごとに同じインデックスを持つので、安全です。
　WallBox::OnCreate()関数では、上記のMyObject::OnCreate()関数を呼び出した後、個別のクリエイト処理を行います。 
		//メッシュ
		m_mesh = BaseMesh::CreateCube(pCommandList, 1.0f);
		//テクスチャ
		auto texFile = App::GetRelativeAssetsDir() + L"wall.jpg";
		m_texture = BaseTexture::CreateTextureFlomFile(pCommandList, texFile);
　です。メッシュは、大きさ1.0fの立方体であり、テクスチャは壁模様のテクスチャです。
Update()関数
Scene::Update()関数は、以下のような内容です。 
	void Scene::Update(double elapsedTime) {
		for (auto& v : m_myObjectvec) {
			v->OnUpdate(elapsedTime);
		}
		UpdateConstantBuffers();
		CommitConstantBuffers();
	}
　ここでは、 
		for (auto& v : m_myObjectvec) {
			v->OnUpdate(elapsedTime);
		}
　と、それぞれのMyObject::OnUpdate()関数を呼び出します。
　この関数は仮想関数なのでWallBox::OnUpdate()関数もしくはSkyStage::OnUpdate()関数が呼ばれます。
　以下はWallBox::OnUpdate()関数ですが 
	void WallBox::OnUpdate(double elapsedTime) {
		m_totalTime += elapsedTime;
		if (m_totalTime >= XM_2PI) {
			m_totalTime = 0.0;
		}
		XMFLOAT3 f(1, 1, 0);
		auto axis = XMLoadFloat3(&f);
		auto spanQt = XMQuaternionRotationAxis(axis, (float)(elapsedTime * 4));
		XMVECTOR quaternion = bsm::makeXM(m_param.quaternion);
		quaternion = XMQuaternionMultiply(quaternion, spanQt);
		m_param.quaternion = bsm::makeF4(quaternion);
		m_param.position.x = (float)sin(m_totalTime) * 2.0f;
	}
　ここではまずm_totalTimeにelapsedTime（前回のターンから経過時間）を足し、これがXM_2PI以上になったら0.0に初期化しています。これでm_totalTimeをもとにサイン関数（三角関数のサイン）を計算します。
　その後1ターンの差分クォータニオンをspanQtに作成し、現在のquaternionに掛けてます。これで、指定の軸に従ってちょっとだけ回転するようになります。
　また行ったり来たりの処理は、m_totalTimeをもとに 
		m_param.position.x = (float)sin(m_totalTime) * 2.0f;
　と、位置情報のx要素を変更してます。
UpdateConstantBuffers()関数
　Scene::UpdateConstantBuffers()関数では、以下のように各MyObjectのUpdateConstantBuffers()関数を呼び出しています。 
	void Scene::UpdateConstantBuffers()
	{
		for (auto& v : m_myObjectvec) {
			v->UpdateConstantBuffers(this);
		}
	}
　これは共通処理でいいのでMyObject::UpdateConstantBuffers()に記述されています。
　結構大きな関数なので、省略して以下に記述します。 
	void MyObject::UpdateConstantBuffers(Scene* scene) {
		//シーンのコンスタントバッファ
		{
			auto& frameResources = scene->GetFrameResources();
			auto pBaseDevice = BaseDevice::GetBaseDevice();
			auto& viewport = scene->GetViewport();

			auto myCamera = scene->GetMyCamera();
			auto myLightSet = scene->GetLightSet();

			//初期化
			m_constantBuffer = {};
			m_constantBuffer.activeFlg.x = 3;
			m_constantBuffer.activeFlg.y = 1;
			//ワールド行列の設定
			auto world = bsm::affineTransformation(m_param);
			auto view = bsm::makeXMMat(myCamera->GetViewMatrix());
			auto proj = bsm::makeXMMat(myCamera->GetProjMatrix());
			auto worldView = world * view;
			m_constantBuffer.worldViewProj = bsm::makeF4x4(XMMatrixTranspose(XMMatrixMultiply(worldView, proj)));
			//中略
		}
		//シャドウマップのコンスタントバッファ
		{
			auto& frameResources = scene->GetFrameResources();
			auto pBaseDevice = BaseDevice::GetBaseDevice();
			auto& viewport = scene->GetViewport();

			auto myCamera = scene->GetMyCamera();
			myCamera->CalculateMatrix();
			auto myLightSet = scene->GetLightSet();

			m_shadowConstantBuffer = {};

			auto lights = myLightSet;
			auto light = lights->GetMainBaseLight();
			//中略

		}
	}
　このように、シーン用のコンスタントバッファとシャドウマップ用のコンスタントバッファを記述します。
　細かい内容についてはサンプルのコードを参照してください。
CommitConstantBuffers()関数
　この関数も同様、Scene::CommitConstantBuffers()関数では、各MyObjectのCommitConstantBuffers()関数を呼び出します。内容は以下です。シンプルです。 
	void MyObject::CommitConstantBuffers(Scene* scene) {
		auto pCurrentFrameResource = scene->GetCurrentFrameResource();
		//シーン
		memcpy(pCurrentFrameResource->m_baseConstantBufferSetVec[m_constantBufferIndex].m_pBaseConstantBufferWO,
			&m_constantBuffer, sizeof(m_constantBuffer));
		//シャドウマップ
		memcpy(pCurrentFrameResource->m_baseConstantBufferSetVec[m_shadowConstantBufferIndex].m_pBaseConstantBufferWO,
			&m_shadowConstantBuffer, sizeof(m_shadowConstantBuffer));
	}
　ここでは、ターゲットとなるフレームscene->GetCurrentFrameResource()で取得を使って、そのフレームの、該当インデックスのm_pBaseConstantBufferWOに対して、各コンスタントバッファをコピーします。
ShadowPass()関数
　ここでは、シャドウマップの描画処理を記述します。
　以下はScene::ShadowPass()関数です。
　ここでは、シャドウマップが書き込めるように、OMSetRenderTargets()関数で準備しています。 
	void Scene::ShadowPass(ID3D12GraphicsCommandList* pCommandList) {
		// Set necessary state.
		//basecross
		auto rootSignature = RootSignaturePool::GetRootSignature(L"BaseCrossDefault", true);
		pCommandList->SetGraphicsRootSignature(rootSignature.Get());
		ComPtr<ID3D12PipelineState> PNTShadowmapPipelineState
			= PipelineStatePool::GetPipelineState(L"PNTShadowmap", true);
		pCommandList->IASetPrimitiveTopology(D3D_PRIMITIVE_TOPOLOGY_TRIANGLELIST);
		pCommandList->RSSetViewports(1, &m_viewport);
		pCommandList->RSSetScissorRects(1, &m_scissorRect);
		// No render target needed for the shadow pass.
		pCommandList->OMSetRenderTargets(0, nullptr, FALSE, &m_depthDsvs[DepthGenPass::Shadow]);
		//以下は各オブジェクト
		for (auto& v : m_myObjectvec) {
			v->OnShadowDraw(pCommandList);
		}
	}
　その後、各MyObject::OnShadowDraw()関数に処理を任せます。
　MyObjectでは、以下のように記述されます。 
	void MyObject::OnShadowDraw(ID3D12GraphicsCommandList* pCommandList) {
		auto pBaseScene = BaseScene::Get();
		auto& frameResources = pBaseScene->GetFrameResources();
		auto pCurrentFrameResource = pBaseScene->GetCurrentFrameResource();

		//Cbv
		pCommandList->SetGraphicsRootConstantBufferView(pBaseScene->GetGpuSlotID(L"b0"),
			pCurrentFrameResource->m_baseConstantBufferSetVec[m_shadowConstantBufferIndex].m_baseConstantBuffer->GetGPUVirtualAddress()); // Set shadow constant buffer.

		pCommandList->IASetVertexBuffers(0, 1, &m_mesh->GetVertexBufferView());
		pCommandList->IASetIndexBuffer(&m_mesh->GetIndexBufferView());
		pCommandList->DrawIndexedInstanced(m_mesh->GetNumIndices(), 1, 0, 0, 0);
	
	}
　ここではコンスタントバッファを、シェーダーのb0というスロットに渡し、m_meshを描画します。
ScenePass()関数
　Scene::ScenePass()関数では、 
	void Scene::ScenePass(ID3D12GraphicsCommandList* pCommandList)
	{
		auto& viewport = GetViewport();
		auto& scissorRect = GetScissorRect();
		auto depthDsvs = GetDepthDsvs();

		//basecross
		auto rootSignature = RootSignaturePool::GetRootSignature(L"BaseCrossDefault", true);
		pCommandList->SetGraphicsRootSignature(rootSignature.Get());

		pCommandList->IASetPrimitiveTopology(D3D_PRIMITIVE_TOPOLOGY_TRIANGLELIST);
		pCommandList->RSSetViewports(1, &viewport);
		pCommandList->RSSetScissorRects(1, &scissorRect);
		pCommandList->SetGraphicsRootDescriptorTable(GetGpuSlotID(L"t0"), m_depthSrvGpuHandles[DepthGenPass::Shadow]);        // Set the shadow texture as an SRV.
		pCommandList->OMSetRenderTargets(1, &GetCurrentBackBufferRtvCpuHandle(), FALSE, &depthDsvs[DepthGenPass::Scene]);
		//パイプラインステートを得る
		ComPtr<ID3D12PipelineState> defaultShadowPipelineState
			= PipelineStatePool::GetPipelineState(L"BcPNTStaticShadow", true);
		pCommandList->SetPipelineState(defaultShadowPipelineState.Get());
		//Sampler
		UINT index = GetSamplerIndex(L"LinearClamp");
		if (index == UINT_MAX) {
			throw BaseException(
				L"LinearClampサンプラーが特定できません。",
				L"Scene::ScenePass()"
			);
		}
		CD3DX12_GPU_DESCRIPTOR_HANDLE samplerHandle(
			GetSamplerDescriptorHeap()->GetGPUDescriptorHandleForHeapStart(),
			index,
			GetSamplerDescriptorHandleIncrementSize()
		);
		pCommandList->SetGraphicsRootDescriptorTable(GetGpuSlotID(L"s0"), samplerHandle);

		index = GetSamplerIndex(L"ComparisonLinear");
		if (index == UINT_MAX) {
			throw BaseException(
				L"ComparisonLinearサンプラーが特定できません。",
				L"Scene::ScenePass()"
			);
		}
		CD3DX12_GPU_DESCRIPTOR_HANDLE samplerHandle2(
			GetSamplerDescriptorHeap()->GetGPUDescriptorHandleForHeapStart(),
			index,
			GetSamplerDescriptorHandleIncrementSize()
		);
		pCommandList->SetGraphicsRootDescriptorTable(GetGpuSlotID(L"s1"), samplerHandle2);

		for (auto& v : m_myObjectvec) {
			v->OnSceneDraw(pCommandList);
		}
	}
　ここでは、ルートシグネチャ、シャドウマップ、サンプラー、パイプラインステートなどを設定し、各GemaObject::OnSceneDraw()関数を呼びます。
　GemaObject::OnSceneDraw()関数は以下です。 
	void MyObject::OnSceneDraw(ID3D12GraphicsCommandList* pCommandList) {

		auto pBaseScene = BaseScene::Get();
		auto& frameResources = pBaseScene->GetFrameResources();
		auto pCurrentFrameResource = pBaseScene->GetCurrentFrameResource();
		auto pBaseDevice = BaseDevice::GetBaseDevice();
		auto& viewport = pBaseScene->GetViewport();
		auto& scissorRect = pBaseScene->GetScissorRect();
		auto depthDsvs = pBaseScene->GetDepthDsvs();

		//シェーダリソースのハンドルの設定
		CD3DX12_GPU_DESCRIPTOR_HANDLE srvHandle(
			pBaseScene->GetCbvSrvUavDescriptorHeap()->GetGPUDescriptorHandleForHeapStart(),
			m_texture->GetSrvIndex(),
			pBaseScene->GetCbvSrvUavDescriptorHandleIncrementSize()
		);
		pCommandList->SetGraphicsRootDescriptorTable(pBaseScene->GetGpuSlotID(L"t1"), srvHandle);

		//Cbv
		// Set scene constant buffer.
		pCommandList->SetGraphicsRootConstantBufferView(pBaseScene->GetGpuSlotID(L"b0"),
			pCurrentFrameResource->m_baseConstantBufferSetVec[m_constantBufferIndex].m_baseConstantBuffer->GetGPUVirtualAddress()); 
		//Draw
		pCommandList->IASetVertexBuffers(0, 1, &m_mesh->GetVertexBufferView());
		pCommandList->IASetIndexBuffer(&m_mesh->GetIndexBufferView());
		pCommandList->DrawIndexedInstanced(m_mesh->GetNumIndices(), 1, 0, 0, 0);
	}
　ここでは、テクスチャおよびコンスタントバッファをシェーダーのスロットに入れます。
　それで、メッシュを渡して、 
		pCommandList->DrawIndexedInstanced(m_mesh->GetNumIndices(), 1, 0, 0, 0);
　と描画処理をします。
まとめ
　このサンプルでは、シンプルながらもシャドウマップやオブジェクトの動的な変化をさせる、3D描画の内容となっています。
　ここでは、オブジェクトは2つですが、これが100個になっても、大きく処理は変わりません。いろんな種類のゲームオブジェクトを作成するためにはMyObjectクラスを継承すれば作成できます。
